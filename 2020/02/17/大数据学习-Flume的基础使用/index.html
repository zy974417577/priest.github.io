大数据学习-Flume的基础使用

  • 大数据

Flume的定义：

  • Flume是一个分布式的、高可靠的、高可用的将大批量的不同数据源的日志数据收集、聚合、移动到数据中心（HDFS）进行存储的系统。即是日志采集和汇总的工具

  • Logstash、FileBeat是ES栈的日志数据抽取工具，他们和Flume很类似，前者是轻量级、后者是重量级，若项目组使用的是ES栈技术，那完全可以使用Logstash取代Flume。

Flume的版本：

​ Flume的版本分为了ng、og俩个版本，ng是新版本，og就是旧版本。在1.0之后全都是ng版本，0.9之前都是og的

  • NG: 1.x的版本 (N=NEW)
  • OG：0.9.x的版本(O=OLD）

Flume的组成：

source(源端数据采集)：

​ 从哪里收集数据，Flume提供了各种各样的Source、同时还提供了自定义的Source。举例几个比较常用的source。其余的可以查文档，太多记不住。

Avro Source：

​ 监听Avro端口并从外部Avro客户端流接收事件。

Exec Source：

​ 执行一个命令行，cat [named pipe] or tail -F [file]

Taildir Source:

​ 监听一个目录，一旦检测到添加到每个文件中的新数据，就实时地进行监控。如果正在写入新数据，则此源将在写入完成之前重试读取它们。

Kafka Source：

​ 从Kafka主题中读取消息。

Channel(临时存储聚合数据)：

​ 主要用于临时存储的地方，相当于一个水池，主要有三种memory channe、File channel、kafka channe。最常用的memory channel和File channel，生产中channel的数据一定是要监控的，防止sink挂了，撑爆channel。

Memory channel

​ 内存当做临时存储

File channel

​ 文件当做临时存储

Kafka channe

​ Kafka做临时存储

Sink（移动数据到目标端）

​ 数据最后发送到哪里，举例几个常用的Sink。

HDFS Sink

​ 数据最后发送Hadoop上去。

Kafka Sink

​ 数据发送到Kafka上去。

Avro Sink

​ 配合Avro Source一起使用。
