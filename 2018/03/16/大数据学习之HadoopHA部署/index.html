虚拟机准备

准备CentOS 7.2

​ 准备虚拟机镜像,安装过程请看另一篇文章….

设置虚拟机ip

​ 具体内容，请看另一篇文章。。。

关闭防火墙

​ (坑！CentOS 7关闭防火墙命令跟CentOS 6 版本不相同，以下命令是CentOS 7)

​ systemctl status firewalld.service 查看防火墙状态

​ systemctl stop firewalld.service 关闭运行的防火墙关闭运行的防火墙

​ systemctl disable firewalld.service 禁止防火墙服务器

设置主机名称

​ 执行命令： hostname hadoop001

​ vi /etc/sysconfig/network

​ NETWORKING=yes
​ HOSTNAME=hadoop001

ip、与 hostname 绑定(3 台)

​ vim /etc/hosts 添加下面三行

​ 192.168.72.151 hadoop001
​ 192.168.72.152 hadoop002
​ 192.168.72.153 hadoop003

设置三台虚拟机SSH认证

​ $ ssh-keygen -t rsa -P ‘’ -f ~/.ssh/id_rsa

​ $ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

​ $ chmod 0600 ~/.ssh/authorized_keys

​ 先生成公钥、私钥。然后三台节点的公钥全部写到authorized_keys 文件里，然后分发各个机器，最后赋给权限600

安装JDK

​ 解压JDK: tar -zxvf jdk-8u45-linux-x64.gz -C /uer/java

​ 修改JDK权限：chown -R root:root /usr/java/* 特别注意。安装好的JDK权限不是ROOT一定要更改

​ 修改profile文件：vi /etc/profile 添加：

​ export JAVA_HOME=/usr/java/jdk1.8.0_45
​ export HADOOP_HOME=/opt/software/hadoop

安装Zookeeper

上传解压安装：

​ [hadoop@hadoop001 software]# tar -zxvf zookeeper-3.4.5-cdh5.16.2 -C ~/app
​ [hadoop@hadoop001 software]cd ~/app
​ [hadoop@hadoop001 app]# ln -s zookeeper-3.4.5-cdh5.16.2 zookeeper

修改配置文件：

​ [hadoop@hadoop001 app]cd zookeeper/conf
​ [hadoop@hadoop001 conf]# cp zoo_sample.cfg zoo.cfg
​ [hadoop@hadoop001 conf]# vi zoo.cfg
​ 添加： dataDir=/home/hadoop/data/zookeeper
​ server.1=hadoop001:2888:3888
​ server.2=hadoop002:2888:3888
​ server.3=hadoop003:2888:3888
​ [root@hadoop001 conf]# cd ~/
​ [root@hadoop001 #]# mkdir data/zookeeper
​ [root@hadoop001 #]# touch data/zookeeper/myid
​ [root@hadoop001 #]# echo 1 > data/zookeeper/myid
​ hadoop002、hadoop003跟上面配置一样

​ *特别注意！！！切记不可 echo 3>data/myid,将>前后空格保留,否则无法将 3 写入 myid 文件* 、

安装Hadoop HA

下载解压 hadoop-2.8.1.tar.gz

​ [hadoop@hadoop001 soft]$ tar -zxvf hadoop-2.6.0-cdh5.16.2.tar.gz -C ~/app
​ [hadoop@hadoop001 soft]ln -s hadoop-2.6.0-cdh5.16.2.tar.gz hadoop

修改hadoop-env.sh

​ 添加：export JAVA_HOME=”/usr/java/jdk1.8.0_45”

修改core-site.xml

修改hdfs-site,xml

修改mapred-stie.xml

修改yarn-site.xml

修改slaves

创建临时文件夹和分发文件夹

[hadoop@hadoop001 ~]$ mkdir -p /home/hadoop/tmp/hadoop
全部分发到hadoop002、hadoop003

启动集群(第一次系统启动,需要初始化)

启动zk

​ [hadoop@hadoop001 zookeeper]# $ZOOKEEPER_HOME/bin/zkServer.sh start
​ [hadoop@hadoop002 zookeeper]# $ZOOKEEPER_HOME/bin/zkServer.sh start
​ [hadoop@hadoop003 zookeeper]# $ZOOKEEPER_HOME/bin/zkServer.sh start

启动 hadoop(HDFS+YARN)

​ 1.格式化前,先在 journalnode 节点机器上先启动 JournalNode 进程

​ [hadoop@hadoop001 ~]# cd ~/app/hadoop/sbin
​ [hadoop@hadoop001 sbin]# hadoop-daemon.sh start journalnode
​ hadoop002、hadoop003也要启动、

NameNode 格式化

​ [hadoop@hadoop001 bin]# hadoop namenode -format

同步 NameNode 元数据

​ 同步 hadoop001 元数据到 hadoop002主要是:dfs.namenode.name.dir,dfs.namenode.edits.dir 还应该确保共享存储目录下(dfs.namenode.shared.edits.dir ) 包含 NameNode 所有的元数据

​ [hadoop@hadoop001 dfs]$ cd /home/hadoop/data/dfs
​ [hadoop@hadoop001 dfs] scp -r ./name hadoop@hadoop002:$pwd

初始化 ZFCK

​ [hadoop@hadoop001 bin]# hdfs zkfc -formatZK

启动 HDFS 分布式存储系统 跟yarn

​ [hadoop@hadoop001 sbin]# start-all.sh
